## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Spoofing | `/api/trades` POST (emotion fields) | mitigate | `requireAuth()` already enforces authentication; userId from JWT, never from body |
| T-3-02 | Tampering | Emotion enum injection | mitigate | Prisma schema enum restricts values; zod validation rejects invalid strings; server validates before DB write |
| T-3-03 | Information Disclosure | Emotion data leakage across users | mitigate | All queries filtered by `userId` from token; no cross-tenant access |
| T-3-04 | Denial of Service | Large import payloads with new fields | accept | Rate limiting deferred to Phase 4; CSV import handled per-row, existing granularity OK |
| T-3-05 | Elevation of Privilege | Fatigue level out-of-range (e.g. 100) | mitigate | zod refine bounds (1–10); server-side validation before Prisma create |

Existing protections (SQLi via Prisma, XSS via sanitization elsewhere) remain unchanged.
</threat_model>
