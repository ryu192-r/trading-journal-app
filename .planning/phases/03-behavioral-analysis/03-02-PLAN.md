# Behavioral heuristics from FEATURES.md (research)

Revenge trading patterns to detect:
- Cluster: 3+ trades within 45 minutes after a losing trade
- End-of-day rush: 2+ trades in last 30 min of session (15:00–15:30 IST)
- Monday spike: trades on Monday > 1.5× individual's daily average (minimum 3 trades Monday)

Overtrading patterns:
- Excessive: daily trade count > avg + 2σ over lookback window (7–30 days)
- EOD rush and Monday spike (same definitions as revenge)

Emotion correlation (BEHV-05):
- Pre-trade emotion → outcome (win rate, avg R)
- During-trade emotion → outcome
- Post-trade emotion → outcome
- Filter out emotion states with < 3 trades (statistically insignificant)

Technical notes:
- IST timezone handling: entryDate stored as UTC in DB; convert to Asia/Kolkata for EOD detection
- Use Prisma for queries: `prisma.trade.findMany({ where: { userId }, orderBy: { entryDate: 'asc' } })`
- 30-day lookback is default but parameterized

    ```typescript
    import { prisma } from '@/lib/db'

    export type RevengePattern = {
      pattern: 'CLUSTER' | 'EOD_RUSH' | 'MONDAY_SPIKE'
      date: string // YYYY-MM-DD
      description: string
      severity: 'LOW' | 'MEDIUM' | 'HIGH'
    }

    export type OvertradingPattern = {
      pattern: 'EXCESSIVE' | 'EOD_RUSH' | 'MONDAY_SPIKE'
      date: string
      tradeCount: number
      avgBenchmark: number
      severity: 'LOW' | 'MEDIUM' | 'HIGH'
    }

    export type EmotionStats = {
      pre: Record<EmotionType, { count: number; winRate: number; avgR: number; profitFactor: number }>
      during: Record<EmotionType, {...}>
      post: Record<EmotionType, {...}>
    }
    ```

    Implement:

    `detectRevengeTrading(userId: string, lookbackDays = 7): Promise<RevengePattern[]>`  
    - Fetch trades from last `lookbackDays` including only closed trades (exitPrice exists), order by entryDate asc
    - Scan sequentially:
      * For each losing trade (pnl < 0 or (exitPrice < entryPrice for LONG)), start window: next 45 min
      * Count trades entering within that window; if ≥ 3 → push CLUSTER pattern with date = losing trade entryDate, severity HIGH
      * For each calendar day: count trades after 15:00 IST (entryDate converted)—if ≥ 2 → EOD_RUSH (MEDIUM)
      * Group by ISO weekday; compute average trades/day over lookback; if Monday count > 1.5×avg and Monday count ≥ 3 → MONDAY_SPIKE (LOW)
    - Return sorted by date desc

    `detectOvertrading(userId: string, lookbackDays = 30): Promise<OvertradingPattern[]>`  
    - Group trades by DATE(entryDate) over lookback
    - Compute mean μ and standard deviation σ of daily counts
      * σ = sqrt(Σ (x-μ)² / n)
    - For each day: if count > μ + 2σ → EXCESSIVE (HIGH)
    - Also check EOD_RUSH and MONDAY_SPIKE same way (reuse logic but report separately)
    - Return patterns array

    `computeEmotionCorrelation(userId: string): Promise<EmotionStats>`  
    - For each dimension pre/during/post:
      - Query all closed trades with that emotion set
      - Group by emotion value
      - For each group: total trades, wins, avgR (mean of R-multiple computed via `computeRMultiple` from calculator.ts), profitFactor = sum(winPnls)/abs(sum(lossPnls))
      - Only include groups with count ≥ 3
      - Return as Record<EmotionType, stats>

    Helper: `toIST(date: Date): Date` using Intl or timezone offset (entryDate stored UTC, IST = UTC+5:30). Use simple `new Date(date.getTime() + 5.5*60*60*1000)`.

    Note: R-multiple computation requires stopPrice. If missing, skip trade from emotion groups or treat as unknown risk (skip). That's acceptable; BEHV-05 focus on complete trades.

<task type="auto">
  <name>Task 2: Create behavioral analytics API endpoints</name>
  <files>src/app/api/analytics/behaviors/route.ts</files>
  <action>
    Create directory `src/app/api/analytics/behaviors` and file `route.ts`:

    ```typescript
    import { NextRequest, NextResponse } from 'next/server'
    import { requireAuth, getUserIdFromToken } from '@/middleware/auth'
    import { detectRevengeTrading, detectOvertrading } from '@/lib/analytics/behavioral'
    import { computeEmotionCorrelation } from '@/lib/analytics/behavioral'

    export async function GET(request: NextRequest) {
      await requireAuth()
      const { pathname } = new URL(request.url)
      const userId = await getUserIdFromToken()
      if (!userId) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })

      if (pathname.includes('patterns')) {
        const revenge = await detectRevengeTrading(userId)
        const overtrading = await detectOvertrading(userId)
        return NextResponse.json({ patterns: [...revenge, ...overtrading].sort(/* by date desc */) })
      }

      if (pathname.includes('emotion-correlation')) {
        const correlation = await computeEmotionCorrelation(userId)
        return NextResponse.json({ correlation })
      }

      return NextResponse.json({ error: 'Not found' }, { status: 404 })
    }
    ```

    Route structure: this single route handles both endpoints via pathname branching (like /api/analytics/r-multiples). Alternative: separate files `patterns/route.ts` and `emotion-correlation/route.ts`; simpler for this phase to combine in one file. Use conditional check on request.url path.

    Both responses return JSON; errors caught by try/catch returning 500.

    Do NOT forget requireAuth at top.
